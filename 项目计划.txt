Emby 挂载播放器 - 需求规格说明书
一、项目概述
1.1 项目名称
EyPlayer
1.2 项目目标
开发一款Windows 桌面端轻量级 Emby 媒体库管理与播放软件，专注于美观的媒体展示和流畅的本地播放器调用体验，无需复杂的转码和服务器功能，仅作为 Emby 服务器的前端展示与播放入口。
1.3 目标用户
拥有个人 Emby 媒体服务器，且媒体库以 .strm 流媒体文件为主，追求简洁美观界面和原生播放器播放体验的用户。
1.4 开发环境与技术栈

    开发工具：VS Code + Claude Code 插件
    推荐技术栈：Electron + Vue 3 + Vite（跨平台桌面应用，开发门槛低，Claude Code 支持度高）
    运行环境：Windows 10/11 64 位

二、核心功能需求
2.1 Emby 媒体库连接与挂载

    ✅ 支持输入 Emby 服务器地址、API 密钥和用户凭证进行连接
    ✅ 自动同步 Emby 服务器上的所有媒体库（电影、电视剧、动漫等）
    ✅ 支持多个 Emby 服务器账号的添加与切换
    ✅ 连接状态实时显示，断网时保留已缓存的媒体信息

2.2 媒体库分类与展示优化

    ✅ 自动识别 Emby 服务器中已有的媒体库分类（国产剧、日韩剧、电影、动漫等）
    ✅ 支持用户手动调整分类的显示顺序和是否显示
    ✅ 每个分类以独立的横向滚动列表形式展示（类似 Netflix / 网易爆米花）
    ✅ 支持按最新添加、评分、上映时间等维度对每个分类进行排序

2.3 STRM 文件专属播放支持

    ✅ 完美解析 Emby 服务器中的 .strm 流媒体文件
    ✅ 自动提取 .strm 文件中的真实播放地址并传递给本地播放器
    ✅ 支持播放进度同步回 Emby 服务器（在看状态、播放进度）
    ✅ 支持标记为已看 / 未看，收藏功能同步到 Emby 服务器

2.4 本地播放器调用系统

    ✅ 默认播放器：MPV 播放器（优先保证解码能力和性能）
    ✅ 可选播放器：预留 PotPlayer、VLC 播放器的调用接口
    ✅ 支持为不同媒体类型（电影、电视剧）设置不同的默认播放器
    ✅ 自动检测系统中已安装的播放器，无需手动配置路径
    ✅ 支持自定义播放器启动参数（针对高级用户）

2.5 硬件解码与播放性能

    ✅ 自动启用 MPV/PotPlayer/VLC 的硬件解码功能（NVIDIA、AMD、Intel 核显均支持）
    ✅ 播放前自动检测并提示最佳解码方案
    ✅ 支持 4K、HDR 视频的流畅播放（依赖本地播放器能力）

2.6 海报与媒体信息本地缓存

    ✅ 自动缓存所有影视海报、缩略图、背景图到本地磁盘
    ✅ 支持设置缓存大小上限，超过时自动清理最久未使用的缓存
    ✅ 支持手动清理全部缓存
    ✅ 下次打开软件时无需重新下载海报，秒开显示

2.7 基础媒体管理功能

    ✅ 搜索功能：支持按影视名称、演员、导演搜索整个媒体库
    ✅ "在看" 列表：自动同步 Emby 服务器的在看内容，显示播放进度
    ✅ 收藏功能：支持收藏喜欢的影视，单独展示
    ✅ 影视详情页：显示海报、简介、评分、演员列表、集数信息

三、界面设计需求
3.1 整体设计风格

    ✅ 核心风格：苹果 iOS/macOS 设计语言，简约、精致、圆角设计
    ✅ 默认主题：白色毛玻璃效果，影视海报可作为背景透过毛玻璃显示
    ✅ 配色方案：提供 4-5 种马卡龙配色主题（浅粉、浅蓝、浅绿、浅紫、米黄）
    ✅ 圆角设计：所有卡片、按钮、弹窗均使用大圆角（16px-24px）
    ✅ 动画效果：页面切换、按钮点击、弹窗出现均添加流畅的过渡动画

3.2 顶部导航栏设计

    ✅ 位置：页面顶部居中，横向排列
    ✅ 内容：包含 5 个核心按钮（媒体、在看、资源、设置、搜索）
    ✅ 按钮样式：图标 + 文字组合，图标仿照 iOS 系统图标风格
    ✅ 交互特性：
        页面向下滑动时自动隐藏导航栏
        页面向上滑动时自动呼出导航栏
        鼠标悬停在顶部时也可呼出导航栏

3.3 主页布局

    ✅ 顶部区域：Emby 媒体库内容随机推荐轮播（展示 3-5 部热门 / 最新影视）
    ✅ 中部区域：按分类横向排列的影视列表（国产剧、日韩剧、电影等）
    ✅ 每个分类显示标题 + "查看全部" 按钮
    ✅ 影视卡片：圆角海报 + 影视名称 + 评分 / 年份信息
    ✅ 鼠标悬停在海报上时显示放大效果和简要信息

3.4 设置页面设计

    ✅ 左侧：设置分类列表（图标 + 文字）
    ✅ 右侧：对应分类的设置选项
    ✅ 完全仿照 iOS 设置页面的布局和交互
    ✅ 主要设置分类：
        服务器设置（添加 / 编辑 Emby 服务器）
        播放器设置（默认播放器、启动参数）
        外观设置（主题切换、毛玻璃透明度、圆角大小）
        缓存设置（缓存大小、清理缓存）
        关于页面

3.5 其他界面要求

    ✅ 影视详情页：大背景图（毛玻璃效果）+ 海报 + 详细信息 + 播放按钮
    ✅ 搜索页面：搜索框 + 搜索结果分类展示
    ✅ 加载状态：简约的加载动画，避免空白页面
    ✅ 错误提示：友好的错误提示信息，指导用户解决问题

四、非功能需求
4.1 性能要求

    软件启动时间 < 3 秒（已缓存海报情况下）
    媒体库加载时间 < 5 秒（首次连接除外）
    内存占用 < 200MB（正常使用状态）
    无明显卡顿和掉帧现象

4.2 兼容性要求

    支持 Windows 10 1903 及以上版本
    支持 Windows 11 所有版本
    兼容 Emby Server 4.5 及以上版本

4.3 易用性要求

    无需复杂配置，输入 Emby 服务器信息即可使用
    界面直观，操作符合 Windows 用户习惯
    提供简单的首次使用引导

4.4 可维护性要求

    代码结构清晰，模块化设计
    关键代码添加注释
    配置文件使用 JSON 格式，易于修改
